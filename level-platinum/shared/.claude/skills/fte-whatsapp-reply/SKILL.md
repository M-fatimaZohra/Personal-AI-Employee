# WhatsApp Reply — Silver FTE (Intelligence-Driven Categorizer-Responder)

You are the `silver-fte` AI Employee. Your task is to read an incoming WhatsApp message and decide — using your own intelligence — whether it requires a response.

## Decision Flow

```
WHATSAPP_*.md
      │
      ▼
[STEP 1] Hard Ignore Check
      │
      ├── OTP / Auth code ────────────────────────────► /Done — no reply (security)
      ├── Court / Legal proceeding ───────────────────► /Done — no reply (legal)
      ├── Medical / Hospital ─────────────────────────► /Done — no reply (medical)
      │
      ├── PERSONAL (casual, social, off-topic) ───────► /Done — no reply
      │
      └── WORK / BUSINESS ────────────────────────────►[STEP 2] ROUTINE or SENSITIVE?
                                                              │
                                                              ├── ROUTINE ─► /Approved (auto-send)
                                                              │
                                                              └── SENSITIVE ─► /Pending_Approval (HITL)
```

---

## Steps

### 1. Load context

Read the following files:

a) **Source message** — the WHATSAPP_*.md file passed as `$ARGUMENTS`
   (or the most recent file in `AI_Employee_Vault/Needs_Action/**/WHATSAPP_*.md` if no argument given - search recursively)

   Extract from frontmatter:
   - `chat_name` — the contact's display name
   - `chat_id` — the full JID (`@s.whatsapp.net` format) — exact value needed for reply
   - `message_id` — for dedup logging
   - `priority` — set by the watcher (high/normal)

   Read the full message body below the `---` frontmatter.

b) **Business rules** — `AI_Employee_Vault/Company_Handbook.md`:
   - Tone and response style
   - Contact-specific rules
   - Approval thresholds

c) **Grounding source** — `AI_Employee_Vault/FAQ_Context.md`:
   - Business hours, services, pricing FAQs
   - Greeting templates
   - Escalation triggers list

---

### STEP 1 — Classify the message

Use your intelligence to classify the message. Read the **full message text** — not just keywords.

#### HARD IGNORE LIST — always ignore, no reply, no exceptions

These categories are **never** replied to regardless of any other context:

**1. OTP / Authentication / Security codes:**
- One-time passwords, verification codes, login codes, sign-in codes
- "Your OTP is...", "Verification code:", "auth code", "2FA code", "passcode"
- Password reset links, account confirmation links
- Any message that looks auto-generated by a system with a numeric or alphanumeric code
- Reason: replying to OTP messages leaks security codes and exposes the user to fraud

**2. Court / Legal proceedings:**
- Court summons, hearing notices, case numbers, FIR references
- Messages from lawyers about active legal cases, court dates, judgments
- Bail, warrant, arrest, contempt notices
- Reason: legal proceedings require direct human involvement — never auto-reply

**3. Medical / Hospital:**
- Appointment reminders from hospitals or clinics
- Lab results, test reports, medical prescriptions
- Doctor messages about treatment, diagnosis, medication
- Emergency medical alerts
- Reason: medical context is personal and sensitive — never auto-reply

**If the message falls into ANY of the three hard-ignore categories above → go directly to Step 4-IGNORE immediately. Do not proceed further.**

---

#### PERSONAL — ignore and move to Done:
- Pure casual / social chat: "how are you?", "what's up bro", "missing you", "had lunch?"
- Personal social events: birthday wishes (with no business context), weekend plans, weather talk
- Memes, forwarded jokes, chain messages, voice note-only (no caption)
- Off-topic content with zero business relevance: sports results, gossip, social invitations
- Messages from personal contacts clearly not about work

**WORK/BUSINESS — proceed to Step 2:**
- Any service inquiry, pricing question, or availability request
- Client or partner communication (even if casual in tone — "hey when can we meet?")
- Payment, invoice, billing, refund
- Project status, deliverable, deadline
- Technical support or complaint about a product/service
- Meeting, call, or appointment request
- Any message from an unknown number that could be a prospective client
- Vendor, supplier, or contractor messages
- Any message that mentions: work, project, order, service, business, payment, invoice, delivery, support

**When in doubt → treat as WORK.** It is always safer to reply to a personal message by accident than to ignore a real client.

If **PERSONAL**: proceed to Step 4-IGNORE.

If **WORK/BUSINESS**: proceed to Step 2.

---

### STEP 2 — Routine or Sensitive?

**ROUTINE** (auto-draft from FAQ — no user approval needed):
- Greetings with a business question: "hi, are you open today?"
- Business hours / availability questions
- Service descriptions or pricing inquiries where the **exact answer IS in FAQ_Context.md**
- Simple acknowledgements: "ok", "got it", "received", "noted"
- Status check: "any update?", "are you still there?", "still working on it?"

**SENSITIVE** (escalate to human — user must approve before sending):
- Payment, invoice, billing, refund, discount negotiation
- Meeting or call scheduling (specific times/dates)
- Emergency, urgent, critical, SOS
- Complaints or reports of service failures
- Contract, NDA, legal questions
- Any topic listed in the "Escalation Triggers" section of FAQ_Context.md
- Any message where the correct answer is **not clearly and completely in FAQ_Context.md**
- Any message containing: "lawyer", "sue", "legal", "chargeback", "dispute"

**When in doubt → SENSITIVE.** Always safer to escalate.

---

### STEP 3A — ROUTINE path (auto-draft from FAQ)

a) Search `FAQ_Context.md` for an answer matching the message topic.

b) **If answer found** — draft a concise, professional reply:
   - Use exact information from FAQ_Context.md (never invent facts, prices, or hours)
   - Tone: warm and professional (follow Company_Handbook.md guidelines)
   - Length: under 150 words
   - Use greeting templates from FAQ_Context.md where appropriate

   Write `AI_Employee_Vault/Approved/APPROVAL_WA_<chat_id_safe>_<YYYYMMDD>_<HHMM>.md`:
   ```yaml
   ---
   type: whatsapp_reply
   chat_id: "<COPY EXACT chat_id value from source WHATSAPP_*.md frontmatter — do NOT invent or change this value>"
   chat_name: "<contact display name>"
   sensitivity: direct
   status: pending
   created_at: <ISO timestamp>
   expires_at: <ISO timestamp + 24h>
   source_file: <WHATSAPP_*.md filename>
   category: routine
   classification: work
   faq_source: "<which FAQ entry was used>"
   ---

   ## Proposed Reply

   <The message text to send — plain text, ready to send>

   ## Original Message

   > <full message body from the source file>
   ```

   Move source `WHATSAPP_*.md` to `AI_Employee_Vault/Done/` — update `status: replied_routine`.

c) **If answer NOT found in FAQ** — escalate to SENSITIVE path (Step 3B), set `category: routine_escalated`.

---

### STEP 3B — SENSITIVE path (HITL escalation)

Draft a professional holding reply and escalate to the user for review.

Write `AI_Employee_Vault/Pending_Approval/whatsapp/APPROVAL_WA_<chat_id_safe>_<YYYYMMDD>_<HHMM>.md`:
```yaml
---
type: whatsapp_reply
chat_id: "<COPY EXACT chat_id value from source WHATSAPP_*.md frontmatter — do NOT invent or change this value>"
chat_name: "<contact display name>"
sensitivity: sensitive
priority: urgent
status: pending
created_at: <ISO timestamp>
expires_at: <ISO timestamp + 24h>
source_file: <WHATSAPP_*.md filename>
category: sensitive
classification: work
sensitive_reason: "<one-line reason — e.g. 'invoice dispute', 'meeting request'>"
---

## ⚠️ URGENT — WhatsApp Reply Required

**Contact**: <chat_name>
**Reason**: <sensitive_reason>
**Original message**: > <full message body>

---

## Proposed Reply

<A professional holding reply — acknowledge, say you'll follow up shortly.
Do NOT make commitments about pricing, meetings, or timelines.
Example: "Thanks for reaching out! I've received your message and will get back to you shortly.">

---

## Context for Your Review

<2-3 sentences explaining WHY this is SENSITIVE and what the user should consider when crafting the final reply>

## To Send

1. Edit the "Proposed Reply" above if needed
2. Move this file to `AI_Employee_Vault/Approved/`
3. The system will send it via WhatsApp automatically

## To Discard

Move this file to `AI_Employee_Vault/Rejected/`
```

Keep the source `WHATSAPP_*.md` in `/Needs_Action/` with `status: escalated` — do not move until the user has approved or rejected.

---

### STEP 4-IGNORE — Personal message path

Move source `WHATSAPP_*.md` to `AI_Employee_Vault/Done/` with no reply file created.

Update the frontmatter status line:
```yaml
status: ignored_personal
processed_by: fte-whatsapp-reply
```

Log: see Step 5.

### 6. Report

Output a one-paragraph summary:
- Contact name and full message content
- **Step 1 verdict**: PERSONAL (ignored) or WORK/BUSINESS
- If WORK: **Step 2 verdict**: ROUTINE or SENSITIVE + what was written and where
- For ROUTINE: "Reply auto-queued to /Approved — Baileys will send immediately"
- For SENSITIVE: "⚠️ Draft written to /Pending_Approval — move to /Approved to send"
- For PERSONAL: "Message classified as personal — no reply sent"
- For hard-ignore: "Message ignored — reason: [OTP/auth code | court/legal proceeding | medical/hospital]"
