|------|--------|

All vault subdirectories except `Dashboard.md` and `Company_Handbook.md` are gitignored:

| Path | Reason |

## PHR Files (history/prompts/003-gold-tier/)

PHR (Prompt History Records) are development session logs generated by the SDD workflow.

The entire `history/prompts/003-gold-tier/` directory is **gitignored** (root `.gitignore`).

**Why**: Live testing sessions produced PHR files containing real client names (e.g. from invoice workflows), real email addresses, Odoo partner IDs, and Gmail message IDs from end-to-end pipeline tests. These are internal audit logs, not code artifacts.

---

## No Hardcoded Credentials

All code reads credentials exclusively from environment variables and `.secrets/` files. No tokens, passwords, or API keys appear in committed code.

**The one exception to watch**: `.env.example` must never contain real values — only placeholders. Verify before committing.

---

## Odoo API Key Scope

The Odoo API key grants full Administrator access to the Odoo instance. Treat it with the same care as a database password:

- Store only in `.env` (gitignored)
- Rotate if `.env` is accidentally shared
- The MCP Odoo server validates the key on startup — if invalid, tools refuse to register
